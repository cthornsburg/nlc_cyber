How to Submit (CTF Writeups + Lab Requests)

This page is for students who have limited or no experience with Git or GitHub.

What you can submit

CTF writeups (allowed)

  • How you approached the challenge (your thought process)
  • Steps that are reproducible
  • Tools/commands used (sanitized)
  • What you learned
Do NOT include: flags, tokens, passwords, API keys, sensitive school/client data, or anything that violates competition rules.

Lab requests (allowed)

  • Request a new lab topic
  • Report a bug or ask for clarification on an existing lab
  • Suggest a dataset (must be legal to share)

Two ways to submit (pick one)

  • Option A (recommended): Use the GitHub website (no installs)
  • Option B (optional): Use Git from your computer

Before you start

  1. Create a GitHub account: github.com/signup
  2. Open the repo: github.com/cthornsburg/nlc_cyber
  3. Read contribution rules: CONTRIBUTING.md

Option A — Submitting via the GitHub website

A1) Lab request (or bug/clarification)

Use a GitHub Issue (think of it like a ticket).

  1. Open the repo: cthornsburg/nlc_cyber
  2. Click Issues
  3. Click New issue
  4. Pick the best template (if shown): new-lab or fix-errata
  5. Fill it out and submit

What to include in a lab request

  • Skill/topic (example: “Windows event log triage”)
  • Difficulty level (intro/intermediate/advanced)
  • What you want to be able to do at the end
  • References/links (optional)

A2) CTF writeup

Step 1 — Fork the repository

A fork is your own copy of the repo on GitHub.

  1. Go to: cthornsburg/nlc_cyber
  2. Click Fork (top right)
  3. Create the fork

Step 2 — Add your writeup file

  1. In your fork, go to ctf/writeups/ (create it if needed)
  2. Click Add file → Create new file
  3. Name it like: ctf/writeups/YYYY-MM-DD_platform_challenge-name.md
  4. Paste your writeup
  5. Commit message example: Add writeup: picoCTF web challenge (sanitized)
  6. Click Commit new file

Step 3 — Open a Pull Request (PR)

A Pull Request is how you ask to merge your changes into the main repo.

  1. In your fork, click Pull requests → New pull request
  2. Confirm base repo: cthornsburg/nlc_cyber and base branch: master
  3. Create the pull request
  4. In the description, include: challenge name/platform/category + confirmation “No flags/secrets included”

Writeup template (copy/paste)

# [Platform] — [Challenge Name]

## Summary
1–3 sentences: what the challenge was about.

## What I tried (and why)
- Step-by-step, with reasoning.

## Solution (sanitized)
- Commands, tools, key observations.
- No flags, no secrets.

## Lessons learned
- What you’d do faster next time.
- Common pitfalls.

## References
- Links to docs/posts you used.

Option B — Submitting with Git (optional)

This is not required. Use this once you’re comfortable.

git --version

git clone git@github.com:<your-username>/nlc_cyber.git
cd nlc_cyber

git checkout -b add-writeup-<topic>
# add your file(s)

git add ctf/writeups/your-file.md
git commit -m "Add writeup: ..."
git push -u origin add-writeup-<topic>
